§2

Retriever Architecture

TF-IDF → Agentic → Hybrid + RAGAS

왜 세 가지를 비교하는가

v1 만으로는 어휘 표면이 다르거나 의미가 분산된 질문을 풀 수 없고, v2 의 휴리스틱은 *알고 있는* 오류 유형만 잡는다. v3 의 임베딩·하이브리드·정량 평가가 두 한계를 해결하는지는 *동일한 30문항* 위에서만 정량적으로 답할 수 있다.

점수의 의미

본 페이지의 v3 점수는 RRF 융합값이다 — 어휘·BM25·n-gram 세 랭킹의 *역수 합*. 절대 의미는 없으나 같은 질의 내 비교에 사용된다.

Golden Dataset · Retrieval Recall

30 questions · L1 / L2 / L3 · recall@5

RAGAS 의 Context Precision·Recall 은 (질문, 정답, 검색된 컨텍스트, *정답 컨텍스트*) 의 4-tuple 을 입력으로 받는다. 따라서 골든셋의 각 항목은 단순한 (Q, A) 가 아니라 relevant_article_ids 까지 갖추어 검색 단계의 회수율을 정량 측정할 수 있다. 본 페이지에서는 그 중 가장 직관적인 지표인 recall@5 를 v1·v2·v3 에 대해 측정한다.

recall@5 의 한계

recall@5 는 검색 단계의 회수율만 본다. 답변의 사실성(Faithfulness) 과 컨텍스트의 정확성(Precision) 은 Python 의 ragas_runner.py 가 Claude 판정으로 산출한다.

§5

Method Notes

tokenization · TF-IDF · BM25 · RRF · heuristics

Tokenizer. 한국어 조사·어미 19종을 단순 제거한 뒤 영문은 lowercase 로 통일한다. 한 토큰의 길이 하한은 2자이며, 조사 제거 전후 두 버전을 모두 보존한다. 동의어 사전 19쌍이 한국어·영어·축약형을 묶어 표면 차이를 흡수한다.

TF-IDF (v1). 코퍼스를 tf · idf 로 벡터화하고 질의·문서 사이의 cosine similarity 로 점수화한다. idf(t) = log((N+1)/(df+1)) + 1. 결과는 Top-3.

BM25. 평균 문서 길이 보정과 saturating frequency 를 적용한 sparse 점수 — k1=1.5, b=0.75. v3 hybrid 의 sparse 신호.

Char n-gram Jaccard. n=3 의 문자 n-gram 집합 간 Jaccard 유사도. 어휘 단위 토큰이 일치하지 않더라도 형태적 유사성을 포착해 *임베딩의 의미적 유사성* 을 부분적으로 근사한다.

RRF. 세 랭킹의 역수 점수를 합산한다 — score(d) = Σ 1 / (k + ranki(d)) with k=60. 점수 스케일이 다른 세 신호를 안정적으로 융합한다.

v2 휴리스틱. 세 유형. (i) 개념 대체 — 진보성을 creativity/originality 로 치환하는 답, 영업비밀에 '유일성' 을 요건으로 넣는 답을 검출. (ii) 도메인 혼동 — 반도체 강제실시 질문에 도하 선언·의약품 공공보건 논리를 끌어오는 답을 검출. (iii) 기간 혼동 — 답변의 연수와 회수된 조항의 허용 연수가 어긋나는 경우를 검출.

v3 충실성 근사. 본 페이지는 답변 토큰의 어떤 비율이 회수된 컨텍스트에 등장하는지를 lexical overlap 으로 계산한다 — RAGAS Faithfulness 의 *상한 근사*. 정식 Faithfulness 는 답변을 원자 명제로 분해하고 각 명제의 컨텍스트 함의를 LLM 판정으로 검증한다.

왜 RRF 인가

세 신호의 점수 스케일이 다르므로 가중합(weighted sum) 은 보정이 어렵다. RRF 는 *랭크* 만 사용하므로 스케일 무관, 하이퍼파라미터는 k 하나뿐이다. k=60 은 IR 문헌의 디폴트.

왜 char n-gram 인가

BGE-M3 같은 모델 없이 브라우저에서 의미 유사도를 *완전히* 재현할 수는 없다. 그러나 문자 단위 n-gram 은 토큰 분리가 다른 한·영 혼합 쿼리에서 형태 유사성을 잡아 어휘 일치 실패를 완화한다 — 임베딩의 완벽한 대체가 아닌 *부분 근사*.
